◆ AcquisitionStopMode

enum peak::core::AcquisitionStopMode
strong

The enum holding the possible acquisition stop modes, for DataStream::StopAcquisition().

See GenTL ACQ_STOP_FLAGS.

Enumerator
Default 

Stops the acquisition engine when the currently running tasks like filling a buffer are completed (default behavior).

Kill 

Stop the acquisition engine immediately. In case this results in a partially filled buffer the Producer will return the buffer through the regular mechanism to the user, indicating through the info function of that buffer that this buffer is not complete.

Custom 

Starting value for GenTL Producer custom IDs which are implementation specific.

◆ DataStreamFlushMode

enum peak::core::DataStreamFlushMode
strong

Operation modes for DataStream::Flush() method.

See GenTL ACQ_QUEUE_TYPE.

Enumerator
InputPoolToOutputQueue 

Flushes the buffers from the input pool to the output buffer queue and if necessary adds entries in the "New Buffer" event data queue. The buffers currently being filled are not affected by this operation.

DiscardOutputQueue 

Discards all buffers in the output buffer queue and if necessary remove the entries from the event data queue.

AllToInputPool 

Puts all buffers in the input pool. This is including those in the output buffer queue and the ones which are currently being filled and discard entries in the event data queue.

UnqueuedToInputPool 

Puts all buffers that are neither in the input pool nor being currently filled nor in the output buffer queue in the input pool.

DiscardAll 

Discards all buffers in the input pool and the buffers in the output queue including buffers currently being filled so that no buffer is bound to any internal mechanism and all buffers may be revoked or requeued.

Custom 

Starting value for GenTL Producer custom IDs which are implementation specific.

◆ FirmwareUpdateVersionStyle

enum peak::core::FirmwareUpdateVersionStyle
strong

The style of the version value.

This is needed to actually interpret and sort the versions. Using this information an update software is able to inform the user if an update would actually be a downgrade or if it was already applied to the device.

Enumerator
Dotted 

The version consists of any number of parts separated by dots. If a part consists of decimal characters only, it is compared numerically, otherwise it is compared using strcmp(). This leads to the following ordering: 1.1.a < 1.10.a < 1.10.b <1.10.b.a

Semantic 

The style as specified in 'Semantic Versioning 2.0.0' ( http://semver.org/ ).
